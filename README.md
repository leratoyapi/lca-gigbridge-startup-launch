##  About GigBridge

**GigBridge** is a specialized, dual-sided freelance portal that bridges the gap between early-career talent seeking portfolio-building experience and small businesses needing agile, cost-effective support.

---

## Problem Statement

Traditional platforms present a **"catch-22"** where entry-level talent needs experience just to get experience. This manifests through:

- **High Barriers to Entry:** Job listings regularly demand 2–3 years of professional history for "entry-level" roles.
- **Flawed Freelance Algorithms:** Established platforms utilize bidding systems that favor top-rated sellers, leaving students completely hidden.
- **Prohibitive Costs:** Small startups and entrepreneurs are priced out of standard agency rates, preventing them from getting tech assistance.

---

##  Mission Statement

> To engineer a specialized, dual-sided freelance portal that bridges the gap between early-career talent seeking portfolio-building experience and small businesses needing agile, cost-effective support.

---

##  Target Audience

- University students
- Recent graduates
- Young professionals
- Small businesses
- Entrepreneurs

---

##  Unique Selling Proposition (USP)

> Unlike LinkedIn or Upwork, GigBridge offers a curated, supportive ecosystem with zero barriers to entry, focusing exclusively on entry-level micro-gigs and automated skill-matchmaking that pairs a client's project needs directly with the exact coding modules a student is currently mastering.

### Workflow Rules

1. **No direct commits to `main`** – All work must go through a pull request.
2. Each team member works on their own branch.
3. Branches must be kept up to date with `main`.
4. Pull Requests require **at least one team review** before merging.
5. The GitHub Lead (Lerato) is responsible for approving and merging.

### Workflow Steps

```bash
# 1. Create your branch from main
git checkout main
git pull origin main
git checkout -b your-name-dev

# 2. Make changes and commit
git add .
git commit -m "Describe your changes"

# 3. Push your branch
git push -u origin your-name-dev

# 4. Create a Pull Request on GitHub

# 5. Request a review from a teammate

# 6. After approval, merge into main

# 7. Delete your branch (optional)
